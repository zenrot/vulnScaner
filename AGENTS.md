# AGENTS.md

## Документация проекта

Вся проектная AI-документация находится в `ai_project_knowledge/`.

Перед **любой** задачей AI-агент (Codex/Claude) обязан:

1. Открыть `ai_project_knowledge/README.md`.
2. По таблице **"Быстрая Навигация По Задачам"** выбрать только файлы, которые прямо касаются текущей задачи.
3. Прочитать выбранные тематические файлы один раз.
4. В первом шаге кратко написать, какие файлы из `ai_project_knowledge/` прочитаны и почему.

`README.md` читается всегда. Остальные файлы читаются только если они прямо нужны для задачи.

Если для задачи достаточно только `README.md`, так и написать:

> Прочитал `ai_project_knowledge/README.md`. Тематические файлы не открывал, потому что задача не требует дополнительного проектного контекста.

## Ограничение На Чтение

Документация читается один раз в начале задачи.

Запрещено:

- перечитывать уже открытые файлы “для надежности”, “чтобы не забыть”, “для перепроверки контекста”;
- читать всю папку `ai_project_knowledge/` целиком без явного запроса;
- воспринимать README как чеклист обязательного чтения всех файлов;
- открывать файлы из `ai_project_knowledge/`, которые не относятся к текущей задаче;
- читать старые task-log файлы в начале новой задачи вместо постоянной документации.

Использовать уже прочитанное из контекста текущей сессии.

## Когда Можно Перечитать Файл

Повторное чтение уже открытого файла разрешено только если:

- пользователь явно попросил перечитать или перепроверить документацию;
- задача изменилась и стала относиться к другой теме;
- файл был изменен во время текущей задачи;
- найдено противоречие между документацией и кодом/dev-стендом;
- нужен точный факт, команда, путь или цитата из конкретного файла, без которого нельзя корректно продолжить.

Если файл перечитывается, кратко написать какой файл и зачем.

## Контекстный Бюджет

По умолчанию для одной задачи:

- `ai_project_knowledge/README.md` — обязательно, 1 раз;
- тематические файлы — только релевантные, обычно 0–3 файла;
- больше 3 тематических файлов открывать только если задача действительно межсервисная, архитектурная или исследовательская;
- при нехватке контекста сначала искать точечно по коду или конкретному файлу, а не расширять чтение всей базы знаний.

## Лог Текущей Задачи

Во время каждой новой задачи AI-агент (Codex/Claude) должен вести краткий append-only лог действий внутри `ai_project_knowledge/current_task/` или `ai_project_knowledge/logs/`.

Цель лога — сохранить факты и результаты, чтобы позже пополнять постоянную базу знаний и не расследовать одно и то же заново.

Лог должен быть коротким:

- для маленькой задачи — несколько строк;
- для большой задачи — краткие этапы и результаты;
- писать только факты: что смотрел, что менял, какие команды запускал, какие новые устойчивые факты нашел;
- не писать chain-of-thought, длинные рассуждения и временные гипотезы без результата;
- не копировать секреты, токены, пароли, приватные ключи, полные env/config values.

Важно:

- старые task-log файлы не являются стартовой документацией;
- не читать все старые логи в начале новой задачи;
- читать старый лог только если пользователь просит продолжить конкретную прошлую задачу или слить логи в постоянную документацию;
- постоянным источником истины остаются файлы верхнего уровня `ai_project_knowledge/*.md` и `ai_project_knowledge/services/*.md`.

Если найден новый важный факт, в конце лога пометить его как кандидат на перенос в постоянную документацию.

Постоянные markdown-файлы обновлять только если пользователь явно попросил слить или обновить документацию.

## Правила Безопасности

- Не менять бизнес-код без явного запроса.
- Не запускать миграции без явного разрешения.
- Не перезапускать контейнеры без явного разрешения.
- Не выводить и не сохранять секреты, токены, пароли, приватные ключи.
- Если документация расходится с кодом или dev-стендом — явно отметить расхождение.
